# A1 — How do I style a component in this repo?

## Use design tokens, not hardcoded values

Every component has a `styles.scss` file (e.g. `src/button/styles.scss`). Styles are written in SCSS and must use design tokens from the `awsui` namespace — never hardcode colors, spacing, or font sizes.

```scss
@use '../internal/styles/tokens' as awsui;

// Spacing
padding-block: awsui.$space-s;
padding-inline: awsui.$space-button-horizontal;
gap: awsui.$space-xs;

// Colors
color: awsui.$color-text-body-default;
background: awsui.$color-background-container-content;
border-color: awsui.$color-border-control-default;

// Typography
font-size: awsui.$font-size-body-m;
line-height: awsui.$line-height-body-m;
font-family: awsui.$font-family-base;
```

## Typography mixins

Import the shared styles module and use the typography mixins instead of setting font properties manually:

```scss
@use '../internal/styles' as styles;

.my-element {
  @include styles.font-body-m;       // standard body text
  @include styles.font-body-s;       // small body text
  @include styles.font-heading-xs;   // heading xs through xl
  @include styles.font-heading-s;
  @include styles.font-heading-m;
  @include styles.font-heading-l;
  @include styles.font-heading-xl;
  @include styles.font-display-l;    // large display text
  @include styles.font-button;       // button text weight + smoothing
  @include styles.font-label;        // form label style
}
```

## Spacing tokens

Spacing uses a scale from xxxs to xxl. Vertical spacing has scaled variants that adapt to context:

- Horizontal: `awsui.$space-xxxs`, `$space-xxs`, `$space-xs`, `$space-s`, `$space-m`, `$space-l`, `$space-xl`, `$space-xxl`
- Vertical (scaled): `awsui.$space-scaled-s`, `$space-scaled-m`, `$space-scaled-l`, etc.

## Key rules from the style guide

1. **Use CSS logical properties only** — no `left`/`right`/`top`/`bottom`/`width`/`height`:

```scss
// ✅ correct
padding-inline-start: awsui.$space-s;
margin-block-end: awsui.$space-m;
inline-size: 100%;

// ❌ wrong
padding-left: awsui.$space-s;
margin-bottom: awsui.$space-m;
width: 100%;
```

2. **No outer margins on root elements** — parent components manage spacing.
3. **No descendant combinators** (`.a .b`) — they break CSS scoping. Use direct child or class selectors.
4. **Wrap animations** in the `with-motion` mixin so motion can be toggled.
5. For **RTL-aware logic** beyond CSS logical properties, use the `with-direction` mixin in SCSS or the direction helpers from `@cloudscape-design/component-toolkit/internal` in TypeScript.
