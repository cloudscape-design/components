## Component file structure

Using `button` as a representative example, each component lives in `src/<component-name>/` with this layout:

```
src/button/
├── index.tsx          # Public entry point — thin wrapper around internal.tsx
├── internal.tsx       # Actual implementation (used for internal composition)
├── interfaces.ts      # Props interface (ButtonProps) with JSDoc for API docs
├── styles.scss        # Component styles (design tokens, logical properties)
├── style.tsx          # Generated/runtime style helpers
├── constants.scss     # SCSS constants/variables
├── icon-helper.tsx    # Component-specific helpers
├── analytics-metadata/  # Analytics integration
├── test-classes/      # Test-specific CSS selectors
├── __tests__/         # Unit tests
└── __integ__/         # Integration tests
```

## Key conventions to follow

**Public vs Internal split:**
- `index.tsx` is the public API — it wraps the internal component, applies `useBaseComponent`, `getBaseProps`, `applyDisplayName`, and passes props through. It should not add behavior.
- `internal.tsx` has the real implementation. Other components can import this directly for composition.

**Props & interfaces (`interfaces.ts`):**
- Name the interface `${ComponentName}Props`, namespace sub-types under it (e.g. `ButtonProps.Variant`)
- Use `ReadonlyArray<T>` for arrays, type aliases for unions (no inline unions)
- Return type must be `JSX.Element` (not null or arrays — breaks doc generation)
- JSDoc comments on props become the public API docs

**Events:**
- All `on*` props must use `CancelableEventHandler<Detail>` or `NonCancelableEventHandler<Detail>` — the build enforces this

**Controllable state:**
- Build the controlled version first, then wrap with `useControllable` to support both controlled and uncontrolled usage

**Styling:**
- Use design tokens, not hardcoded values
- Use CSS logical properties only (`inline-start`, `block-size`, etc.) — no `left`/`right`/`top`/`bottom`/`width`/`height`. This is required for RTL support.
- No descendant combinators (`.a .b`) — breaks CSS scoping
- No outer margins on root elements
- Wrap animations in the `with-motion` mixin

**I18n:**
- Centralize translatable strings under a skippable `i18nStrings` prop

**Shared utilities:**
- Check `src/internal/` before creating new hooks or helpers — there's a lot of shared infrastructure there (`useControllable`, `useForwardFocus`, base component hooks, event utilities, etc.)

**Testing:**
- Unit tests go in `__tests__/`, integration tests in `__integ__/`
- See `docs/WRITING_TESTS.md` and `docs/RUNNING_TESTS.md` for test utilities and commands

**Dependencies:**
- Must support React 16.8+ and the latest 3 major versions of Chrome/Firefox/Edge. No global state, ESM preferred.
