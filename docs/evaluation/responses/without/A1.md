# A1 — How do I style a component in this repo?

## Prompt

> How do I style a component in this repo? What's the right way to apply spacing, colors, and typography?

## Response

Here's how styling works in this repo:

### SCSS with design tokens

Every component has a `styles.scss` file that uses Cloudscape's design token system. You never hardcode colors, spacing, or font values — you reference tokens instead.

#### 1. Import tokens

```scss
@use '../internal/styles/tokens' as awsui;
@use '../internal/styles' as styles;
```

`tokens.scss` forwards from `awsui:tokens`, which is the generated token set from the `style-dictionary/` directory.
You access them as `awsui.$token-name`.

#### 2. Spacing

Use spacing tokens like `awsui.$space-*`:

```scss
padding-block: awsui.$space-scaled-s;
padding-inline: awsui.$space-l;
margin-inline-end: awsui.$space-xxs;
```

Tokens include: `$space-xxxs`, `$space-xxs`, `$space-xs`, `$space-s`, `$space-m`, `$space-l`, `$space-xl`, `$space-xxl`, and scaled variants that adapt to density mode. Component-specific spacing tokens also exist (e.g., `$space-button-horizontal`, `$space-container-horizontal`).

#### 3. Colors

Use color tokens — never raw hex/rgb values:

```scss
color: awsui.$color-text-body-default;
background: awsui.$color-background-container-content;
border-color: awsui.$color-border-divider-default;
```

Colors support light/dark mode automatically through the token system. The definitions live in `style-dictionary/classic/colors.ts` and `style-dictionary/visual-refresh/colors.ts`.

#### 4. Typography

Use the typography mixins from `src/internal/styles/typography/mixins.scss`:

```scss
@include styles.font-body-m;
@include styles.font-heading-s;
@include styles.font-button;
```

Available mixins: `font-body-s`, `font-body-m`, `font-heading-xs` through `font-heading-xl`, `font-display-l`, `font-button`, `font-label`, `font-panel-header`, `font-code-inline`, `font-code-block`.

For individual properties, use tokens directly:

```scss
font-size: awsui.$font-size-heading-m;
font-weight: awsui.$font-weight-heading-s;
line-height: awsui.$line-height-heading-m;
font-family: awsui.$font-family-base;
```

#### 5. Stylelint rules to know

The `.stylelintrc` enforces several constraints in `src/`:

- **No shorthand properties**: `border`, `margin`, `padding` are disallowed — use logical properties like `padding-block`, `padding-inline`, `margin-inline-start`, `border-block-width`, etc.
- **Use logical properties**: enforced by `csstools/use-logical` — always use `inline-size` instead of `width`, `block-size` instead of `height`, `inset-inline-start` instead of `left`, etc.
- **No descendant selectors** (space combinator): use direct child `>` or flat class selectors instead.
- **No type selectors**: don't style `div`, `span`, etc. directly — use class names.
- **No custom CSS properties** defined inline (pattern `^(?!.+)`).
- **No motion outside of mixin**: use the motion mixins from `src/internal/styles/motion/`.

#### 6. Putting it together (example pattern)

```scss
@use '../internal/styles' as styles;
@use '../internal/styles/tokens' as awsui;

.root {
  @include styles.styles-reset;
  @include styles.font-body-m;
  color: awsui.$color-text-body-default;
  padding-block: awsui.$space-scaled-s;
  padding-inline: awsui.$space-l;
  border-block-end: awsui.$border-divider-section-width solid awsui.$color-border-divider-default;
  border-start-start-radius: awsui.$border-radius-container;
}
```

The key takeaway: always use tokens for values and logical properties for layout. The style dictionary in `style-dictionary/` defines all available tokens across themes (classic, visual-refresh).
