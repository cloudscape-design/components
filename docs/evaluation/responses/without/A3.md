# A3 — How do I test components in the browser?

## Prompt

> How do I test components in the browser?

## Response

This repo has a built-in dev server. Run:

```sh
npm start
```

This runs two things in parallel:

- `gulp watch` — rebuilds components on file changes
- `webpack serve` — starts a dev server serving the `pages/` directory

Once it's running, open the URL shown in the terminal (typically `http://localhost:8080`). Each component has test pages under `pages/<component-name>/` that you can navigate to in the browser — for example, `http://localhost:8080/#/light/button/simple`.

The `pages/` directory contains demo/test pages for every component (button, table, modal, etc.), which are useful for visually verifying behavior and testing changes interactively.
